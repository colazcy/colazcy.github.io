Codeforces Round #588

A.Marcin and Training Camp

题意：有 \(n\) 个人，第 \(i\) 个人有属性值 \(a_i\)，能力值 \(b_i\)。若 \(\exists k\)，使得 \(a_i\) 的第 \(k\) 位为 \(1\) 而 \(a_j\) 的第 \(k\) 位为 \(0\)，则第 \(i\) 个人瞧不起第 \(j\) 个人（两个人可以互相瞧不起）。

从 \(n\) 个人中选若干个人，在不存在有某个人瞧不起其他所有人的前提下，最大化选出来的人的能力值之和。

\(1 \leq n \leq 7000\)。

\(0 \leq a_i \leq 2^{60}\)。

\(1 \leq b_i \leq 10^9\)。

3s，256M。

若 \(i\) 瞧不起 \(j\)，则连边 \((i,j)\)。

如果存在一个 \(i\)，瞧不起当前集合内所有人，我们就把 \(i\) 删去。重复操作即可。

拿个队列维护一下。

B.Kamil and Making a Stream

题意：给定一棵有根树，每个点有点权 \(x_i\)。记 \(f(u,v)\) 表示简单路径 \((u,v)\) 上所有点点权的 \(\gcd\)。求： \[ \sum_{\text{u 是 v 的祖先}}f(u,v) \] \(2 \leq n \leq 10^5\)。

\(0 \leq x_i \leq 10^{12}\)。

4s，768M。

我们不断对一个数取 \(\gcd\)，那么变化次数不会超过 \(\log\) 次。

因此我们可以倍增预处理往上跳一段的 \(\gcd\)，之后一段一段求和即可。

C.Konrad and Company Evaluation

题意：给定一张 \(n\) 个点 \(m\) 条边的有向图，\(q\) 次操作。每次操作给定一个点 \(u\)，如果有边 \((x,u)\)，则将其改向为 \((u,x)\)。每次操作之后，求有多少个三元组 \((a,b,c)\)，满足有边 \((a,b),(b,c)\)。

\(n,m,q \leq 10^5\)。

4s，256M。

记一个点的入度为 \(in\)，出度为 \(out\)，那么答案为 \(\sum_u in[u]out[u]\)。

每次操作暴力修改边的方向，均摊复杂度在根号级别。势能分析。

若图方便，使用 std::set 或std::unordered_set 实现无法通过。

D.Wojtek and Card Tricks

题意：给定 \(n\) 个长为 \(k\) 的置换。记 \(f(l,r)\) 表示 \([l,r]\) 中的置换形成的置换群大小。求 \(\sum_{l=1}^n\sum_{r=l}^nf(l,r)\)。

\(1 \leq n \leq 2\times 10^5\)。

\(1 \leq k \leq 5\)。

3.5s，256M。

当我们添加一个置换时，如果它已经在原来的置换群当中，那么不会产生任何影响。否则根据群论中的拉格朗日定理，新的置换群的大小一定是原置换群大小的倍数。

\(k! \leq 120\)，因此我们可以直接暴力。从右往左，维护每一种置换最靠左的出现位置。对于每个 \(l\)，\(f(l,r)\) 只可能在这些位置发生变化。而变化次数不会超过 \(\log k!\) 次。

需要注意的是，当一个置换已经在当前置换群当中时，不要重复加入，这样才能保证复杂度。

E1.Marek and Matching (easy version)

题意：有一张左右部均为 \(n\) 个点的完全二分图，给定每条边存在的概率，求整张二分图存在完美匹配的概率。

\(n \leq 6\)。

7s，256M。

对于 \(n < 6\) 我们可以将图补全至 \(n = 6\)。

此时一共有 \(36\) 条边，考虑折半。即考虑右部的前 \(3\) 个和后 \(3\) 个点与左部 \(6\) 个点之间的匹配情况，之后合并。一共可能有 \(\binom{6}{3} = 20\) 种可能匹配。可以对于 \(2^{18}\) 种可能连边情况暴力计算。

直接计算略微麻烦，考虑补集转化。即求不存在完美匹配的概率。

记 \(P_a(S)\)，表示右部的前 \(3\) 个点，与左部的三元组集合 \(S\) 都存在完美匹配的概率。

记 \(P_b(S)\)，表示右部的后 \(3\) 个点，若三元组 \((a,b,c) \in S\)，则其与左部另外三个点存在完美匹配的概率。

容易发现，若 \(A \cap B=\emptyset\)，则 \(P_a(A)P_b(B)\) 代表一种不存在完美匹配的方案。

因此我们对 \(P_b\) 做一次子集前缀和得到 \(P_b'\)。\(1-\sum_xP_a(x)P_b'(\overline x)\) 即为答案。

E2.Marek and Matching (hard version)

题意：与 E1 相同。

\(n \leq 7\)。

15s，256M。

与上面的思路相似。我们依次考虑右侧每个点 \(i\)，维护一个 \(f[S]\)（\(S\) 是点集的集合）。若点集 \(P \in S\)，那么右侧前 \(i\) 个点与左侧点集 \(P\) 存在完美匹配的概率。

左侧只可能有 \(2^7=128\) 种点集，可以使用 \(128\) 位整数存储。\(2^7\) 暴力枚举右侧点 \(i\) 与左侧点的连边方案，进行转移。

实际上的状态数很少，足以通过。

F.Mateusz and Escape Room

题意：有 \(n\) 个位置排成一个环，第 \(i\) 个位置上一开始有 \(a_i\) 枚硬币。给定 \(n\) 个区间 \([l_i,r_i]\)。每次操作，你可以将一枚硬币，从一个位置移动到和它相邻的某个位置。

求最小的操作次数，使得 \(\forall i\)，最终的硬币数 \(a'_i \in [l_i,r_i]\) 。

\(n,a_i,l_i,r_i \leq 35000\)。

7s，256M。

以另一种形式描述问题，记位置 \(i\) 向位置 \(i + 1\) 传递 \(x_i\) 枚硬币。

要求 \(\forall i \in[2,n] \quad l_i \leq x_{i-1}-x_i+a_i\leq r_i\)，并且 \(l_1 \leq x_n-x_1+a_1 \leq r_1\)。

求 \(\sum |x_i|\) 的最小值。

引理：记 \(f(x)\)，表示 \(x_1 = x\) 的前提下，\(\sum |x_i|\) 的最小值。那么 \(f(x)\) 是一个下凸函数。证明。

首先原问题可以转化为上下界最小费用流问题。而在容量均为整数的图中，即使我们允许流量为实数，我们也一定能够找到一个最优的流，使得流量均为整数。因此我们允许 \(x_i\) 取任意实数不会影响答案。（也就是扩展定义域）。

而如果 \(p\) 是一组合法的 \(x_i\)，\(q\) 是一组合法的 \(x_i\)。那么 \(r_i = \frac{p_i+q_i}{2}\) 也是一组合法的 \(x_i\)。代入即可证明。

由于 \(|x|\) 是一个凸函数，且 \(x_i\) 之间相互独立，那么一定有 \(f(\frac{p+q}{2}) \leq \frac{f(p)+f(q)}{2}\)。

所以我们可以三分求出 \(f(x)\) 的最小值。

现在问题变为，当 \(x_1\) 给定时，如何求出 \(\sum |x_i|\) 的最小值。

容易发现，我们要维护一个凸函数 \(f\)，支持以下操作：

  • \(f \leftarrow f'\)，其中 \(f'(x) = \min_{t=x-l}^{x+r}f(t)\)。
  • \(f \leftarrow f+g\)，其中 \(g\) 也是凸函数（其实就是 \(|x|\)）。
  • 求 \(\min_{x=l}^rf(x)\)。

直接 Slope Trick 维护就可以了。第一种操作对应着，将斜率为 \(0\) 的那一段的左部和右部，分别向左边和右边平移若干距离。

中途维护只关心斜率为 \(0\) 的那一段，那直接对顶堆，打整体标记就行了。
