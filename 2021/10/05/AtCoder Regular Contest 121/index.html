A.2nd Greatest Distance

题意：给定 \(n\) 个点 \((x_i,y_i)\)，对于 \(\frac{n(n-1)}{2}\) 个点对，求它们的切比雪夫距离的次大值。

\(3 \leq n \leq 2 \times 10^5\)。

2s，1024M。

对于一个点，能产生的切比雪夫距离最大的点对，是它和 \(x,y\) 最小 / 最大的点形成的点对。

我们将这 \(4n\) 个点对去重即可。

B.RGB Matching

题意：有 \(2n\) 条狗，每条狗有可爱值 \(a\)，颜色 \(c \in \{1,2,3\}\)。现在要将它们分为 \(n\) 对。如果 \(i\) 和 \(j\) 颜色一样不需要代价，否则需要付出 \(|a_i-a_j|\) 的代价。求配对代价的最小值。

\(1 \leq n \leq 10^5\)。

\(1 \leq a_i \leq 10^{15}\)。

2s，1024M。

首先一共有偶数条狗，那么对于三类颜色，一定是两奇一偶，或者三偶。

对于三类颜色都有偶数条狗，直接内部两两配对即可。

否则，奇数集合与奇数集合配对，或者两个奇数集合都和偶数集合配对。剩下的可以集合内部两两配对。

奇数集合与奇数集合配对，即从两个集合中各选一个数，使得它们之间的差的绝对值尽量小。这个可以通过找前驱后继来完成。但有更简单的实现方式，直接把两个集合丢一起排序，取相邻数看是否来自不同集合。

两个奇数集合和偶数集合配对。也可以通过找前驱后继的方式完成。也可以通过丢一起排序来完成。

有可能出现两个奇数集合和偶数集合中同一个元素配对的情况，但是这样不影响最终答案。因为这样一定不优于奇数集合和奇数集合配对。

C.Odd Even Sort

题意：给定排列 \(p\)，第奇数次操作，可以将奇数 \(x\) 与 \(x + 1\) 位置的元素交换。第偶数次操作，可以将偶数 \(x\) 与 \(x + 1\) 位置的元素交换。在 \(n^2\) 次操作内，将排列 \(p\) 排好序。\(t\) 组询问。

\(1 \leq t \leq 250\)。

\(2 \leq n \leq 500,\sum n \leq 500\)。

2s，1024M。

\(n^2\) 次操作，最多 \(\frac{n(n-1)}{2}\) 对逆序对可能比较有提示性。

那么如果当前操作我们可以将逆序对减少 \(1\) 我们直接交换。

否则我们尝试交换一对元素，使得另一种操作可以将逆序对减少。

如果找不到这样的一对元素？

那么此时 \(n\) 一定是偶数，并且最后四个元素相当于排列 \(1 2 4 3\)，我们交换最后一对能交换的元素。规约为排序排列 \(1 4 2 3\)，手玩发现可以利用上述过程直接将这四个排好序。

D.1 or 2

题意：给定一个长为 \(n\) 的序列，每次操作可以取出一个数或者两个数，写下它们的和。最小化写下的数的最大值与最小值的差。

\(1 \leq n \leq 5000\)。

\(-10^9 \leq a_i \leq 10^9\)。

2s，1024M。

如果每次操作只能取出两个数并写下它们的和，那么我们每次取最大值和最小值即可。

注意到只取一个数相当于再取一个 \(0\)。所以我们枚举往原序列中添加多少个 \(0\)，再按照上述方法贪心即可。

E.Directed Tree

题意：给定一棵以 \(1\) 为根的树。求有多少个排列 \(p\)，满足 \(\forall i,p_i\) 在树上不是 \(i\) 的祖先。

\(1 \leq n \leq 2000\)。

2s，1024M。

要求不是祖先不好算，容斥一波。

我们现在要求有 \(s\) 个位置，满足 \(p_i\) 在树上是 \(i\) 的祖先的方案数。

可以使用树形背包来计数，我们记 \(f[u][s]\)，表示只考虑子树 \(u\) 内的点，有 \(s\) 个点，满足 \(p_i\) 在树上是 \(i\) 的祖先的方案数。

那么合并子树就是树形背包，利用子树大小优化枚举上界可以做到 \(\Theta(n^2)\) 的复杂度。

现在我们考虑 \(u\) 这个点所产生的贡献，也就是在子树 \(u\) 内选一个点 \(x\) 使得 \(p_x = u\)。

假设我们从只合并了子树的 \(f'\) 转移到最终的 \(f\)。

那么 \(f[u][s] = f'[u][s] + (siz[u] - s)f'[u][s - 1]\).

最终答案为 \(\sum_s (-1)^s f[1][s](n-s)!\)

F.Logical Operations on Tree

题意：给定一棵 \(n\) 个点的树，将每条边标记为 AND 或者 OR。将每个点标记为 \(0\) 或者 \(1\)。每次选择一条边，按照上面标记的操作，进行缩点。如果最后能得到单个权值为 \(1\) 的节点，那么这种标记方案是合法的。求有多少种合法的标记方案。

\(1 \leq n \leq 2 \times 10^5\)。

\(0 \leq a_i \leq 2^{30}\)。

4s，1024M。

对于这种缩点问题，可以从叶子节点考虑。

  • 初始被标记为 \(0\)，操作为 AND。那么我们肯定不能在最后去执行 AND 操作，立马执行一定更优。
  • 初始被标记为 \(0\)，操作为 OR，那么这个点没有任何影响。
  • 初始被标记为 \(1\)，操作为 AND，那么这个点没有任何影响。
  • 初始被标记为 \(1\)，操作为 OR，那么无论其他点怎么取，都是合法标记方案。

综上讨论，AND 操作立刻执行是更优的。

由于存在不管其他点怎么取都合法的情况，为了方便，我们强制不出现第四种情况，计算不合法的方案数。

而此时，对叶子节点进行 OR 操作不会产生影响。因此我们可以从下往上进行树形 dp。

记 \(f[u][s]\) 表示将子树 \(u\) 缩点，最后得到 \(s\) 的方案数。

转移类似于背包，每次合并一棵子树。注意不能出现新加入子树缩为 \(1\)，而边为 OR 这种情况。

最后答案为 \(2^{2n-1} - f[1][0]\)。
